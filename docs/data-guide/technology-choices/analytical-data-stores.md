# <a name="choosing-an-analytical-data-store-in-azure"></a><span data-ttu-id="cf8c6-102">Escolhendo um armazenamento de dados analíticos no Azure</span><span class="sxs-lookup"><span data-stu-id="cf8c6-102">Choosing an analytical data store in Azure</span></span>

<span data-ttu-id="cf8c6-103">Em uma arquitetura de [Big Data](../big-data/index.md), geralmente, há a necessidade de um armazenamento de dados analíticos que forneça dados processados em um formato estruturado que pode ser consultado com ferramentas analíticas.</span><span class="sxs-lookup"><span data-stu-id="cf8c6-103">In a [big data](../big-data/index.md) architecture, there is often a need for an analytical data store that serves processed data in a structured format that can be queried using analytical tools.</span></span> <span data-ttu-id="cf8c6-104">Os armazenamentos de dados analíticos que dão suporte à consulta de dados de caminho quente e frio são chamados em conjunto de camada de serviço ou armazenamento de serviço de dados.</span><span class="sxs-lookup"><span data-stu-id="cf8c6-104">Analytical data stores that support querying of both hot-path and cold-path data are collectively referred to as the serving layer, or data serving storage.</span></span>

<span data-ttu-id="cf8c6-105">A camada de serviço lida com os dados processados dos caminhos quente e frio.</span><span class="sxs-lookup"><span data-stu-id="cf8c6-105">The serving layer deals with processed data from both the hot path and cold path.</span></span> <span data-ttu-id="cf8c6-106">Na [arquitetura lambda](../big-data/index.md#lambda-architecture), a camada de serviço é subdividida em uma camada de _serviço de velocidade_, que armazena os dados que foram processados de forma incremental e uma camada de _serviço de lote_, que contém a saída processada em lotes.</span><span class="sxs-lookup"><span data-stu-id="cf8c6-106">In the [lambda architecture](../big-data/index.md#lambda-architecture), the serving layer is subdivided into a _speed serving_ layer, which stores data that has been processed incrementally, and a _batch serving_ layer, which contains the batch-processed output.</span></span> <span data-ttu-id="cf8c6-107">A camada de serviço exige suporte forte para leituras aleatórias com baixa latência.</span><span class="sxs-lookup"><span data-stu-id="cf8c6-107">The serving layer requires strong support for random reads with low latency.</span></span> <span data-ttu-id="cf8c6-108">O armazenamento de dados para a camada de velocidade também deve dar suporte a gravações aleatórias, porque o carregamento em lotes dos dados nesse repositório introduzirá atrasos indesejados.</span><span class="sxs-lookup"><span data-stu-id="cf8c6-108">Data storage for the speed layer should also support random writes, because batch loading data into this store would introduce undesired delays.</span></span> <span data-ttu-id="cf8c6-109">Por outro lado, o armazenamento de dados da camada de lote não precisa dar suporte a gravações aleatórias, mas gravações em lotes.</span><span class="sxs-lookup"><span data-stu-id="cf8c6-109">On the other hand, data storage for the batch layer does not need to support random writes, but batch writes instead.</span></span>
