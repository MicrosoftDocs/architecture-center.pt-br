## <a name="mainframe-architecture"></a><span data-ttu-id="83f78-119">Arquitetura de mainframe</span><span class="sxs-lookup"><span data-stu-id="83f78-119">Mainframe architecture</span></span>

<span data-ttu-id="83f78-127">Uma arquitetura típica de mainframe IBM inclui os seguintes componentes comuns:</span><span class="sxs-lookup"><span data-stu-id="83f78-127">A typical IBM mainframe architecture includes these common components:</span></span>

- <span data-ttu-id="83f78-135">**Camada de Banco de Dados**: Um sistema de gerenciador de banco de dados relacional (DBMS) para z/OS é o IBM DD2.</span><span class="sxs-lookup"><span data-stu-id="83f78-135">**Database tier:** A common relational database management system (DBMS) for z/OS is IBM DD2.</span></span> <span data-ttu-id="83f78-136">Ele gerencia as estruturas de dados chamadas *dbspaces* que contêm uma ou mais tabelas e são atribuídas aos pools de armazenamento físicos de conjuntos de dados chamados *dbextents*.</span><span class="sxs-lookup"><span data-stu-id="83f78-136">It manages data structures called *dbspaces* that contain one or more tables and are assigned to storage pools of physical data sets called *dbextents*.</span></span> <span data-ttu-id="83f78-137">Dois componentes de banco de dados importantes são o diretório que identifica os locais de dados nos pools de armazenamento e o log que contém um registro das operações executadas no banco de dados.</span><span class="sxs-lookup"><span data-stu-id="83f78-137">Two important database components are the directory that identifies data locations in the storage pools, and the log that contains a record of operations performed on the database.</span></span> <span data-ttu-id="83f78-138">Há suporte para vários formatos de dados de arquivo simples.</span><span class="sxs-lookup"><span data-stu-id="83f78-138">Various flat-file data formats are supported.</span></span> <span data-ttu-id="83f78-139">Normalmente, o DB2 para z/OS usa os conjuntos de dados do método de acesso de armazenamento virtual (VSAM) para armazenar os dados.</span><span class="sxs-lookup"><span data-stu-id="83f78-139">DB2 for z/OS typically uses virtual storage access method (VSAM) datasets to store the data.</span></span>

- <span data-ttu-id="83f78-143">**LPAR:** Partições lógicas ou LPARs, são usados para dividir os recursos de computação.</span><span class="sxs-lookup"><span data-stu-id="83f78-143">**LPAR:** Logical partitions, or LPARs, are used to divide compute resources.</span></span> <span data-ttu-id="83f78-144">Um mainframe físico é particionado em vários LPARs.</span><span class="sxs-lookup"><span data-stu-id="83f78-144">A physical mainframe is partitioned into multiple LPARs.</span></span>

- <span data-ttu-id="83f78-145">**z/OS:** Um sistema operacional de 64 bits que é mais comumente usado para mainframes IBM.</span><span class="sxs-lookup"><span data-stu-id="83f78-145">**z/OS:** A 64-bit operating system that is most commonly used for IBM mainframes.</span></span>
